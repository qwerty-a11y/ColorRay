# ColorRay

> **A Color-encoded Visible Light Communication System.**  
> **一个使用色彩编码的可见光传输系统。**

---

## 1. 项目概述

### 1.1 项目背景

可见光通信（Visible Light Communication, VLC）是一种新兴的无线通信方式。本项目利用屏幕显示和摄像头拍摄，在无需额外硬件的情况下实现数据传输。该技术适用于离线环境、安全传输及物联网等应用场景。

### 1.2 项目目标

- **软件系统**：开发一套基于自定义彩色二维码的软件系统。
- **传输链路**：实现从 PC 屏幕（发送端）到手机拍摄（接收端）的文件传输。
- **核心指标**：高容量、强鲁棒性、具备容错能力，文件还原成功率 $\ge 90\%$。

---

## 2. 系统架构与技术流程

### 2.1 发送端流程

1. **文件处理**：将待传输文件进行二进制分块。
2. **冗余编码**：引入 RAID 5/6 帧间纠删码，生成校验块。
3. **差错检测**：为每个数据块附加 CRC16 校验位。
4. **二维码生成**：将数据映射为彩色像素，生成自定义彩色二维码图像。
5. **视频合成**：将图像序列合成为视频流并播放。

### 2.2 接收端流程

1. **视频拍摄**：手机拍摄 PC 端的二维码视频流。
2. **帧提取**：从视频中提取每一帧二维码图像。
3. **预处理**：进行定位块识别、透视校正与模块采样。
4. **颜色解码**：利用标准色块进行色彩校准，识别 8 色信息并转换为二进制数据。
5. **校验与恢复**：通过 CRC16 检测错误，利用 RAID 机制恢复丢失或损坏的帧。
6. **文件重组**：合并所有数据块，还原原始文件。

---

## 3. 核心设计规范

### 3.1 二维码设计简介

- **基础规格**：采用版本 29 规格（$133 \times 133$ 模块）。
- **颜色方案**：8 种纯色编码（红、绿、蓝、黄、青、品红、黑、白）。
- **功能图形设计**：
  - **定位**：四角保留“回”字形定位块。
  - **校准**：右下角放置 8 个标准色块，用于实时色彩补偿。
  - **元数据**：三个大定位块附近设有 18 个色块，包含 5 个序号位和 1 个纠删等级位（多向备份）。

### 3.2 冗余与纠错机制

系统基于 **RAID 5/6** 实现帧间纠删，支持三种冗余级别：

| 冗余度 | 分组大小 | RAID 类型 | 容错能力 |
| --- | --- | --- | --- |
| **10%** | 10 块/组 | RAID 5 | 每组可丢 1 块 |
| **20%** | 5 块/组 | RAID 5 | 每组可丢 1 块 |
| **40%** | 5 块/组 | RAID 6 | 每组可丢 2 块 |

> **帧内校验**：每个数据块附带 16 位 CRC 校验。

## 4. 模块分工

| 分工模块 | 主要职责 |
| --- | --- |
| **原始数据包装** | 文件分块、RAID 编码、CRC16 计算 |
| **数据编码** | 将数据块转换为彩色二维码图像，生成视频 |
| **数据解码** | 从视频帧中提取二维码，颜色分类，获取数据块 |
| **还原原始数据** | CRC 校验、RAID 解码、文件重组 |
| **定位变形标准化** | 定位块识别、透视校正、模块采样 |
